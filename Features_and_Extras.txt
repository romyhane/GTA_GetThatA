Bonus Features Implementation:
------------------------------

1. Color Support:
   We implemented full color support using the Windows Console API 
   (SetConsoleTextAttribute).
   - The game includes a Main Menu option [5] to toggle colors ON/OFF.
   - The game works perfectly in both Color and Black & White modes.
   - Different elements (Walls, Players, Bombs, Keys) have distinct colors 
     for better visibility.

2. User Interface (UI):
   - Added ASCII Art for the Main Menu and Instructions.
   - Added a dynamic "Funny Wall Collision" message system that randomizes 
     responses when a player hits a wall.
   - Added a persistent Status Bar for inventory tracking.
   
3. Advanced Switch Mechanics:
   We implemented two distinct types of switches using a single 'Switch' class and a 
   boolean flag ('isLatching'), adding depth to puzzle design:
   - Smart Switch (Momentary): The connected door remains open ONLY as long as 
     a player is physically standing on the switch. Once pressure is removed, the door closes immediately.
	 Color : BROWN always. (you cannot see when its ON - player is stepping on it).
   - Regular Switch (Latching/Permanent): Once activated by stepping on it, 
     the switch permanently toggles to the 'ON' state and the connected door remains open forever, regardless 
	 of player position.
	 Color : OFF - RED , ON - GREEN.
	 
4. Adaptive HUD & Unicode Support:
   We enhanced the Status Bar to support rich visuals while maintaining 
   backward compatibility:
   - Graphic Mode (Color ON): Uses Windows Unicode API (WriteConsoleW) to render 
     actual Heart symbols (♥) for player lives.
   - Safe Mode (Color OFF): Automatically detects the toggle and switches 
     to standard ASCII characters ('L') to prevent display glitches on 
     older terminals or non-Unicode environments.

5. The "Fail Monster" Mechanic:
   We introduced a special enemy type and environment logic in Level 3 
   to increase difficulty and variety:
   - The Fail Monster: A unique entity that patrols the map. Unlike bombs 
     that reduce lives, contact with this monster results in an immediate 
     Game Over (Instant Death).
	 
6. Dynamic Level Scaling (Filesystem API):
   - Integrated the C++ <filesystem> library to dynamically scan the working directory 
     for map files.
   - Maps are stored in a dynamic vector, allowing the engine to support any number 
     of levels seamlessly.
   - Implemented dynamic Dark Room detection by detecting '~' characters at runtime 
     instead of using hardcoded coordinates.

7. Decoupled Riddle System:
   - Completely decoupled the riddle loading logic. All riddles are loaded into a 
     global memory pool.
   - Implemented a cyclic assignment algorithm. Any '?' tile placed on any map 
     automatically pulls the next valid question from the pool.

8. Spatial Coupling for Interactive Objects:
   - Developed a "Spatial Coupling" algorithm for switch-to-door connections.
   - Upon level load, each switch object scans the grid and dynamically links to 
     the nearest Door object using Euclidean distance calculations.
   - This decoupled architecture allows full freedom in map design without backend 
     code changes.

9. Advanced Physics (Spring Mechanics & Priming):
   - Upgraded the physics logic to include a "Priming" phase for Spring tiles.
   - Players enter a locked "charged" state upon contact with a spring (triggering 
     a compression animation).
   - The kinetic launch is only executed when the player explicitly triggers the 
     release command.
	 
10. Command-Line Arguments & Automation (Testing Modes):
    The game engine supports advanced execution modes via command-line arguments 
    to facilitate automated testing and debugging:
    - "-save": Records all game events, random seeds, and player inputs into 
      dedicated tracking files (.steps and .result).
    - "-load": Runs the game in playback mode, automatically executing the 
      recorded keystrokes from the .steps file.
    - "-silent": Runs the engine in "Headless" mode (bypassing all visual 
      rendering and UI delays) for instant automated test validation against 
      expected results.
